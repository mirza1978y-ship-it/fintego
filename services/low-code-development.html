FAQ

Low-code development is an approach to software development in which developers use a drag-and-drop interface to build applications instead of writing code in a code editor. This approach comes with several benefits: 

  • Speeding up the development process, reducing time-to-market as a result 
  • Reducing development costs 
  • Enabling professionals with little to no coding experience to create their own apps or applets 

Low-code tools like OutSystems and Appian allow both professional and citizen developers to build the application’s logic and interface using a drag-and-drop editor and a WYSIWYG (What You See Is What You Get) interface builder. The low-code platform then translates those inputs into the code using pre-written reusable components. 

As low-code tools replace code writing with visual workflows, the component code itself is reused within and across applications. While component reusability has significant advantages – greater speed, efficiency, consistency and scalability – it also limits customization, functionality and integration options. 

Low-code platforms also support a limited range of application types and structures. That’s why selecting the right platform for the chosen business cases is crucial. 

Therefore, the low-code development route is a suitable option for simple, standardized applications that don’t require extensive customizations or custom security or regulatory compliance features. Bear in mind that if your application is performance-intensive due to complex functionality or computing requirements, traditional development may more suitable. 

The types of applications that can be built with low-code tools include: 

  • Customer-facing mobile apps 
  • Online stores 
  • Business process automation apps and applets 
  • Customer relationship management systems 
  • Inventory management systems 

Every low-code environment has its own inherent security protections for applications built with it. Those differ from one tool to another but typically include: 

  • Automated vulnerability and security testing 
  • Basic governance and role-based access controls 
  • Compliance with security standards like SOC2 and ISO 27001 

Low-code applications are secure as long as the tool used to build them applies best practices to the final product. However, remember that you can’t introduce your own security features that go beyond the functionality offered by the low-code tool. Therefore, if you operate in a highly regulated industry, you need to ensure the low-code platform is compliant with relevant regulations (e.g., HIPAA, GDPR). 

Another crucial security consideration is citizen developers’ training. Citizen developers are likely to need guidance to build secure applications with attention to liabilities and interdependencies and test them before deployment. 

Low-code applications can be developed in weeks, as opposed to months in the high-code approach, thanks to component reusability and visual interface-based workflows. This faster time-to-market translates into lower development costs. 

The total cost of ownership for low-code applications is also reduced compared to high-code solutions as it’s easier to maintain, scale and update the application. 

However, you have to choose business use cases wisely for low-code development as not all application types are worth building using the low-code approach. It works best for simple applications with no advanced, complex or unique requirements toward integrations, customizations, functionality or security.
